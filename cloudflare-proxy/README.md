# SceneShot CORS-прокси (Cloudflare Worker)

Крошечный бесплатный прокси, чтобы веб-версия SceneShot могла открывать видео **по ссылке**:
прямые ссылки на файлы + **Dropbox** + **Google Drive**. Браузер не может это сам из-за CORS;
прокси скачивает файл у себя и отдаёт его с нужными заголовками.

**Не покрывает YouTube / TikTok / Instagram** — это не файлы, а страницы с зашифрованными потоками;
им нужен `yt-dlp` на полноценном сервере (это отдельная история, не Worker).

Бесплатный тариф Cloudflare Workers: до 100 000 запросов в день, без карты. Подходит для личного
использования. Очень большие файлы (несколько ГБ) могут упереться в лимиты Worker — для них лучше
скачать файл и загрузить локально.

---

## Деплой за 2 минуты

### Вариант A — через дашборд (без установки)

1. Зайдите на <https://dash.cloudflare.com> → **Workers & Pages** → **Create application** →
   **Create Worker**. Дайте имя (например `sceneshot-proxy`) → **Deploy**.
2. Нажмите **Edit code**, удалите шаблон и вставьте целиком содержимое [`worker.js`](worker.js) →
   **Deploy**.
3. Скопируйте адрес воркера вида `https://sceneshot-proxy.<ваш-сабдомен>.workers.dev`.
4. Откройте сайт SceneShot → **⚙ Настройки → «Прокси для ссылок»** → вставьте этот адрес → «Готово».

> В `worker.js` вверху есть список `ALLOWED_ORIGINS`. Там уже прописан
> `https://nonamezzz322.github.io`. Если у вас другой адрес сайта — впишите свой (иначе прокси
> ответит `403 origin not allowed`).

### Вариант B — через CLI (wrangler)

```sh
npm i -g wrangler
cd cloudflare-proxy
wrangler login          # откроет браузер для входа в Cloudflare
wrangler deploy         # задеплоит worker.js, выведет *.workers.dev URL
```

Затем так же вставьте URL в настройках сайта.

## Как это работает / безопасность

- Прокси отвечает **только** на запросы с разрешённых Origin (`ALLOWED_ORIGINS`) — чужие сайты им
  воспользоваться не смогут (браузер шлёт `Origin` сам).
- Пропускает **только медиа** (не HTML-страницы), чтобы не превратиться в открытый веб-прокси.
- Сам **нормализует** ссылки Dropbox (`?dl=1`) и Google Drive (`/file/d/<id>` → прямая загрузка,
  включая попытку пройти страницу подтверждения для больших файлов — Google периодически её меняет).
- Ничего не логирует и не кэширует (`Cache-Control: no-store`).

Хотите жёстче — добавьте в `worker.js` белый список доменов (разрешать только dropbox/google/ваш CDN).
